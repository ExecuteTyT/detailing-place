# Detailing Place — Стартовый пакет

## Что внутри
```
CLAUDE.md                    ← Мастер-инструкция (Claude Code читает автоматически)
docs/
├── SERVICES_DATA.md         ← Все данные 14 услуг (пакеты, FAQ, цены, SEO)
├── FUNNEL.md                ← Спецификация воронки продаж (6 элементов)
└── sprints/
    ├── sprint-1.md          ← Фундамент: UI, Header, Footer, Panel
    ├── sprint-2.md          ← 11 секций-конструкторов
    ├── sprint-3.md          ← Сборка 18 страниц
    ├── sprint-4.md          ← Воронка: popup, callback, quiz, toast
    ├── sprint-5.md          ← UX-полировка, mobile, Lighthouse
    └── sprint-6.md          ← SEO, аналитика, деплой
```

## Как работать с Claude Code

### Подготовка
1. Установи Claude Code: https://docs.anthropic.com/en/docs/claude-code
2. Открой терминал в папке этого проекта
3. Запусти `claude` (или `claude-code`)

### Спринт 1 (фундамент)
```
Прочитай CLAUDE.md и docs/sprints/sprint-1.md. Выполни все задачи спринта 1 по порядку. Используй данные из docs/SERVICES_DATA.md для constants.ts.
```

### Спринт 2 (секции)
```
Прочитай docs/sprints/sprint-2.md. Создай все 11 секций-компонентов и services.ts. Все данные бери из docs/SERVICES_DATA.md. Каждый компонент принимает данные через props.
```

### Спринт 3 (страницы)
```
Прочитай docs/sprints/sprint-3.md. Собери все 18 страниц из готовых компонентов. Данные — через getService(slug) из services.ts. Каждая страница — generateMetadata() для SEO.
```

### Спринт 4 (воронка)
```
Прочитай docs/sprints/sprint-4.md и docs/FUNNEL.md. Создай все 6 элементов воронки и встрой их в layout/страницы.
```

### Спринт 5 (UX)
```
Прочитай docs/sprints/sprint-5.md. Добавь scroll-анимации, lazy loading, touch-жесты. Проверь каждую страницу на mobile 375px. Добейся Lighthouse >90.
```

### Спринт 6 (запуск)
```
Прочитай docs/sprints/sprint-6.md. SEO metadata для всех страниц, JSON-LD Schema.org, sitemap, Яндекс.Метрика, API route для форм, деплой.
```

## Важные правила
- **Один спринт = один промпт.** Не давай всё сразу
- **Проверяй чеклист** в конце каждого спринта перед переходом к следующему
- **Если что-то не так** — скажи Claude Code конкретно что исправить, он чинит быстро
- **Фото-заглушки:** Claude Code создаст placeholder'ы. Реальные фото заменишь потом
- **TODO в коде:** телефон, адрес, WhatsApp, Метрика ID, CRM webhook — заменишь на реальные данные
